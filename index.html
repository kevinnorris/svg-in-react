How To Use SVGs In React

Why Use SVGs?

* But first, Why?

SVG: 3KB

PNG: 17KB

* Small file sizes that compresses well * Scale to any size without loss of clarity 
							
								.react-logo {
								  fill: #FF0081;
								}
* Easily modified with CSS * To change color update fill property 
							
								.react-logo {
								  fill: #FF0081;
								}
								
								.react-logo circle {
								  fill: #8F2AA3;
								}
* Can target individual parts on an SVG and style them differently 
							
								.react-logo {
								  fill: #FF0081;
								}

								.react-logo path {
								  transform-origin: 50% 50%;
								  animation: rotate 3s linear infinite;
								}
								
								.react-logo circle {
								  fill: #8F2AA3;
								  animation: pulse 1s ease-out alternate infinite;
								}
* Can even animate the individual parts * If you wanted to do any modifications to PNG => image editor => save new file * Animating color changes on PNG not possible

Where can we use SVGs?

Icons

Line Graphics

Logos

* SVGs are good for simple images * For complex images file sizes will be larger * As browser must draw the svg can take long to load

What About React?

* We know we want to use SVGs, how in React * The exmples in this talk assume you are bundling your react project with Webpack

File Paths

              
                {
                  test: /\.svg$/,
                  use: 'file-loader',
                }
              
            
              
                {
                  test: /\.svg$/,
                  use: [
                    {
                      loader: 'url-loader',
                      options: {
                        limit: 8000
                      }
                    }
                  ]
                }
* The easiest way to use SVGs in a React project is with file paths * To get file paths in Webpack you can use: * file-loader: return the files public URL * url-loader: works like file-loader but returns a DataURL if the file is smaller than a byte limit, base64 * Many other options for different encoding of DataURL and other settings

Modifiability?

  • Can't alter SVG property through CSS
  • Must create new file for any changes
  • Cluttered asset folder
* Using svgs with file paths we lost all the cool stuff I showed earlier * You can't even change the color of the whole! * If you want the react svg in a different color you need to make another file, change the fill... * Possibly more requests for resources, if not using url-loader
